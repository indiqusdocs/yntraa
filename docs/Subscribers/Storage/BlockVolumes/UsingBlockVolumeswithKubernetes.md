# Using Block Volumes with Kubernetes

Yntraa Kubernetes also uses the underlying **Block Volumes** service to create persistent volumes for Kubernetes clusters. Yntraa Cloud users can specify the root disk size for each cluster node (control and worker), and the Block Volume service creates the nodes in the cluster with a root disk of the specified size.

:::note
Volume management on cluster nodes is not currently supported on Yntraa Cloud.
:::
